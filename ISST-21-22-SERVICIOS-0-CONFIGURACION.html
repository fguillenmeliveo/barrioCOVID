CONFIGURACIÓN DEL ENTORNO DE PROGRAMACIÓN 

Objetivo

El desarrollo de software se basa en el uso de componentes, bibliotecas y otros artefactos que permiten a los desarrolladores focalizarse en lo importante y específico de la aplicación. La reutilización de elementos predefinidos mejora la productividad, pero a medida que crece el tamaño de la aplicación, se hace más difícil manejar el número de componentes y hay que automatizar su gestión.

Los entornos de programación aportan esta automatización: ayudan a la escritura del código esencial, proporcionan el código auxiliar, comprueban su corrección, compilan, enlazan con otras bibliotecas, las mantienen actualizadas, arrancan las aplicaciones con los elementos de soporte que necesitan, etc.

Este documento ayuda a poner a punto un entorno de complejidad intermedia como el que necesitamos en la asignatura para las sesiones de laboratorio y prácticas. Es necesario tener cierta facilidad en el uso del sistema operativo.

Software necesario

Será preciso tener los siguientes paquetes:

  • visual studio code VSC-(https://code.visualstudio.com/ version Darwin) un entorno completo para programar que soporta la mayoría de actividades para varios lenguajes incluyendo Java. Al instalar VSC, se instalan otras herramientas:
  • java development kit (https://openjdk.java.net/install/ versión 11.0) que contiene el conjunto completo de bibliotecas básicas y herramientas para compilar y ejecutar programas Java. El proceso de instalación de los paquetes Java en VSC pedirá información sobre versiones de jdk instaladas, o lanzará esta instalación
  • maven (https://maven.apache.org/install.html) es el gestor de dependencias y construcción de Java en el que se apoyará VSC. Viene incluido con la instalación de VSC así que no es necesario instalarlo aparte
  • git (https://git-scm.com/ versión 2.30.1) sistema de control de versiones de código abierto
  • (opcional) curl (https://curl.se/ version 7.65) herramienta de línea de órdenes que actúa como cliente textual de cualquier protocolo

Todos los paquetes deben estar presentes en su ordenador antes de empezar a programar; los números de versión pueden no ser exactamente los mismos pero si usa otros considere que las versiones más recientes a menudo provocan errores de compatibilidad, y versiones más antiguas pueden contener elementos obsoletos.

Trabajaremos en varios proyectos, que podrá encontrar en la plataforma GitLab (https://about.gitlab.com/):

  • tfgapi0 para el servicio de persistencia que puede importarse de (https://gitlab.com/juancarlosduenas/tfgapi0.git)
  • tfgweb0 para la aplicación web con frontend de servidor que puede importarse de (https://gitlab.com/juancarlosduenas/tfgweb0.git)

Debe crear una carpeta en su sistema de ficheros para la que tenga todos los derechos de acceso; será donde pondrá todos los ficheros y carpetas.

Visual Studio Code para proyectos Spring Boot

VSC es un entorno de programación ligero y genérico; en (https://www.youtube.com/watch?v=uq4GjRF_860) puede ver cómo prepararlo para Java con Spring Boot.

Primero deberá añadirle algunas extensiones marcando el icono de Extensiones en la parte izquierda de VSD, y escribiendo su nombre en la casilla:

  • Java Extension Pack for programming -incluyendo sus sub-paquetes
  • Spring Boot Extension Pack

En VSC, File->Open folder… vale para abrir su carpeta de desarrollo. Por ejemplo puede importar proyectos de Gitlab a esta carpeta:

  1. abra la ventana de órdenes (cmd-shift-P)
  2. escriba GIT y seleccione the Git: Clone 
  3. cuando lo pida, escriba la URL del repositorio Gitlab y seleccione la carpeta local donde quiere clonar el proyecto (la carpeta local)
  4. File-> Save workspace as… la primera vez para convertir la carpeta de desarrollo en un espacio de trabajo VSC. Cuando añada más proyectos VSC le preguntará si los quiere añadir también a este espacio.

Se sugiere que tome los proyectos de GitLab que se indican, con lo que puede saltarse la siguiente sección, que detalla las operaciones para reproducir un proyecto completo.

Creación de un proyecto Spring Boot en la web

Puede crear el proyecto Spring inicial rellenando el formulario de https://start.spring.io/ y respondiendo a las preguntas como se indica en la siguiente sección, obtendrá un fichero zip para bajar, descomprimir y poner en su carpeta de desarrollo.

Creación de un proyecto Spring Boot en Visual Studio Code

Para crear un proyecto Java-SB desde cero: abra la ventana de órdenes (cmd-shift-P for mac, ctrl-shift-P for windows), escriba spring, y seleccione Spring Initializr: Create a Maven project.

Aparecen algunas preguntas:

  • SB version: 2.4.12
  • project language: Java
  • group-Id: es.upm.dit.apsv
  • artefact-Id: ordermanager
  • packaging type: jar
  • Java version: 11
  • chose dependencies: Spring Boot DevTools, Spring Web
  • place for the project: el nombre de una carpeta en su carpeta de desarrollo.

Le preguntará si quiere añadir el proyecto al espacio de trabajo -conteste yes.

Trabajo con un proyecto Spring Boot en Visual Studio Code

Si tiene proyectos Java en la carpeta los puede importar. También deberá indicar que confía en el envoltorio de Maven. Observará que cada proyecto puede verse desde varios puntos de vista. La vista del espacio de trabajo (en la siguiente figura, un proyecto completo tfgapi1) contiene la carpeta del proyecto, donde verá varias subcarpetas y un fichero muy importante: pom.xml. Incluye toda la información de bibliotecas y versiones del proyecto, lo que lo hace capaz de localizarlas y bajarlas automáticamente para mantener estable el proyecto. Maven también se encarga de crear un fichero jar que se ejecutará en un servidor. Más adelante veremos sus contenidos.

Si el fichero pom.xml no se ha terminado de procesar, puede ocurrir que los proyectos o ficheros no se reconozcan automáticamente como Java. Abra cualquiera de ellos y se reconocerán.

La figura muestra la estructura de carpetas de un proyecto. Usaremos sólo las carpetas src/main/java para poner el código fuente organizado con paquetes de Java, y src/main/resources para poner ficheros de configuración como application.properties, y los ficheros con contenidos estáticos o plantillas HTML; también ahí se pueden poner otros ficheros auxiliares de carga de datos si es preciso. En este ejemplo se aprecia que hay una carpeta frontend en donde se ha puesto código de cliente Javascript (React), para mostrar que se puede tener un proyecto de desarrollo multilenguaje.

Tal como se ha dicho, pom.xml es el fichero que Maven lee para obtener las dependencias y la información de construcción para conseguir tener una aplicación en funcionamiento. pom.xml se crea automáticamente por Spring Initializr con los datos que demos, pero a menudo es necesario añadir más dependencias a otras bibliotecas. Encontrará un fichero pom.xml con dependencias diferentes en cada uno de los proyectos, aunque hay unas pocas comunes -que tienen que ver con SB.

En cuanto se crea el proyecto SB (y en los proyectos que se proporcionan), contiene una aplicación capaz de ejecutarse en un servidor web empotrado en SB. Si marca el botón Run que hay encima del método main, la aplicación arranca como indican las trazas, aunque no tiene nada que hacer. En el Spring Boot dashboard hay también iconos para ejecutar; ponga el ratón en el nombre del proyecto y verá los iconos “play” y “stop”, y las trazas en la consola de depuración.

@SpringBootApplication

public class TFGApplication {

        public static void main(String[] args) {

                SpringApplication.run(TFGApplication.class, args);

        }

Alternativamente, puede ir al Terminal (consola de órdenes del sistema operativo), y lanzar el servicio de esta forma:

./mvnw clean install spring-boot:run -DskipTests=true

Una vez arrancado el servicio, podrá acceder a él a través de un navegador Web, por ejemplo con la siguiente URL  http://localhost:8080. Como es lógico al estar vacío el proyecto (no tiene entidades, ni repositorios ni controladores), no hará nada.

A partir de aquí añadiremos nuestras clases Java y otros elementos del proyecto; si obtiene el menú contextual de src/main/java, entre las opciones verá las de New Package y New Java Class que crean un paquete y una clase Java.

Si ha llegado hasta este punto está preparado para comenzar el desarrollo.
